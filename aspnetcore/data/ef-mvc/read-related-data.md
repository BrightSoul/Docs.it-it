<span data-ttu-id="97238-138">Se si conosce che i dati correlati è necessario per tutte le entità recuperate, il caricamento eager spesso offre prestazioni ottimali, perché è in genere più efficiente delle query separate per ogni entità recuperata una singola query inviate al database.</span><span class="sxs-lookup"><span data-stu-id="97238-138">If you know you need related data for every entity retrieved, eager loading often offers the best performance, because a single query sent to the database is typically more efficient than separate queries for each entity retrieved.</span></span> <span data-ttu-id="97238-139">Ad esempio, si supponga che ogni reparto di dieci corsi correlati.</span><span class="sxs-lookup"><span data-stu-id="97238-139">For example, suppose that each department has ten related courses.</span></span> <span data-ttu-id="97238-140">Caricamento eager di tutti i dati correlati comporta solo una query singola (join) e un singolo round trip al database.</span><span class="sxs-lookup"><span data-stu-id="97238-140">Eager loading of all related data would result in just a single (join) query and a single round trip to the database.</span></span> <span data-ttu-id="97238-141">Una query separata per i corsi per ogni reparto comporta undici round trip al database.</span><span class="sxs-lookup"><span data-stu-id="97238-141">A separate query for courses for each department would result in eleven round trips to the database.</span></span> <span data-ttu-id="97238-142">Il round trip aggiuntivi al database sono particolarmente effetti negativi sulle prestazioni quando la latenza è elevata.</span><span class="sxs-lookup"><span data-stu-id="97238-142">The extra round trips to the database are especially detrimental to performance when latency is high.</span></span>
